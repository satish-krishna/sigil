---
description: Generate a markdown report of bob's fix history and current backlog status
---

Generate a comprehensive markdown report from bob's state files (`.bob/history.json` and `.bob/backlog.json`).

## Usage

`/bob:report [format]`

- `/bob:report` — Full report (history + backlog summary)
- `/bob:report history` — History only
- `/bob:report backlog` — Backlog only

## Step 1: Read State Files

1. Read `.bob/history.json` — if missing, note "No fix history yet"
2. Read `.bob/backlog.json` — if missing, note "No backlog yet"

## Step 2: Generate Report

Produce the following markdown report. Print it directly to the user AND offer to write it to a file (e.g., `.bob/report.md`).

---

## Step 3: Omit Empty Sections

- If there are no entries for a category (e.g., no failed fixes), omit that subsection entirely rather than showing an empty table.
- If history.json doesn't exist, skip the entire "Fix History" section and show "No fix history yet — run `/bob:fix` or wait for the scheduled fix pipeline."
- If backlog.json doesn't exist, skip the "Current Backlog" section and show "No backlog yet — run `/bob:audit` to generate one."

## Step 4: Offer File Output

After printing the report, ask the user:

> "Would you like me to save this report to `.bob/report.md`?"

If yes, write the report to that file. Note: `state/report.md` is not gitignored, so it will be tracked if the user commits it.
